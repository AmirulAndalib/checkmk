# Install Checkmk agent slideout registered agent on wrong site

key        | value
---------- | ---
date       | 2026-07-07T13:41:09.680429+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | wato
level      | 1
compatible | yes

When adding a host whose "Monitored on site" attribute was inherited from its
folder instead of being set explicitly on the host, the "Install Checkmk agent"
slideout offered a registration command pointing at the central site. As a
result the agent was registered on the wrong site.

The slideout, as well as the agent and SNMP connection tests on the host
properties page, now use the effective monitoring site of the host, including
values inherited from the folder.
