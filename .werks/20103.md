# Metric backend: configure metric consolidation inline

key        | value
---------- | ---
date       | 2026-07-06T08:00:00+00:00
version    | 3.0.0b1
class      | feature
edition    | ultimate
component  | wato
level      | 2
compatible | yes

When you configure a metric backend query, either as a graph line in the
custom graph editor or in the special agent rule *Metric backend (custom
query)*, you no longer set how the raw OpenTelemetry samples are
consolidated through the separate *Aggregation lookback* and *Percentile*
input fields. Instead, a single inline control lets you pick the whole
consolidation in one place and shows the current setting as a compact
summary next to the query.

Your existing graph definitions and rules keep working unchanged, so there
is nothing you need to adjust after the upgrade.
