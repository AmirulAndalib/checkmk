# Fix wrong ethernet port speed for Cisco Meraki wireless APs

Previously, the speed reported by the _Interface_ service for Ethernet ports on
Cisco Meraki wireless access points was calculated using a conversion constant
that was off by a factor of 10. This caused the reported speed to always be 10
times too low. This has been fixed.

## Note on upgrading

This bug has been present since the check was introduced, so any host where the
affected _Interface_ service was already discovered has the wrong, too-low speed
persisted as its baseline. After upgrading, the check computes the correct
speed, which no longer matches that old baseline, triggering a one-time
`WARN` with a "Speed changed" notice. Rediscover the affected services to
update the baseline and clear this false `WARN`.
