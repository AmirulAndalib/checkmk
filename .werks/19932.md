# Agent updater: restore description for host proxy environment option

key        | value
---------- | ---

version    | 3.0.0b1
class      | fix
edition    | pro

Previously, when configuring the ruleset _Agent updater (Linux, Windows, Solaris)_
and selecting _Use environment variables from host_ under _Proxy settings_, the form
only showed the raw text "env" instead of explaining what the option does.

This has been fixed. The option again shows the descriptive text: "Agent updater will
follow variables "HTTP_PROXY" and "HTTPS_PROXY" on the host."

Only the displayed text was affected; the configured behavior and stored rules were
always correct.
