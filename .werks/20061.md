# HTML email notifications: show alert handler name and output again

key        | value
---------- | ---
date       | 2026-07-17T07:06:25.137430+00:00
version    | 3.0.0b1
class      | fix
edition    | community
component  | notifications
level      | 1
compatible | yes

When you received an HTML email notification triggered by an alert handler,
the name and output of the alert handler were no longer shown.

Up to Checkmk 2.3 these two fields were part of the notification email. With the
reworked HTML email layout in 2.4 they were accidentally dropped, and no
configuration option could bring them back.

The event overview of an HTML email notification now shows the "Name of alert
handler" and "Output of alert handler" again whenever the notification was
triggered by an alert handler.
